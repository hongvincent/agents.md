# 이 레포지토리를 위한 AGENTS 가이드라인

이 레포지토리는 루트 디렉토리에 위치한 Next.js 애플리케이션을 포함하고 있습니다. 에이전트(예: Codex CLI)와 함께 프로젝트에서 대화형으로 작업할 때 아래 가이드라인을 따르면 개발 경험, 특히 핫 모듈 교체(HMR, Hot Module Replacement)가 원활하게 작동합니다.

## 1. `npm run build`가 **아닌** 개발 서버 사용

* **애플리케이션을 반복 작업할 때는 항상 `npm run dev`(또는 `pnpm dev`, `yarn dev` 등)를 사용하세요.** 이 명령은 핫 리로드가 활성화된 개발 모드에서 Next.js를 시작합니다.
* **에이전트 세션 내에서 `npm run build`를 실행하지 _마세요_.** 프로덕션 빌드 명령을 실행하면 `.next` 폴더가 프로덕션 에셋으로 전환되어 핫 리로드가 비활성화되고 개발 서버가 일관되지 않은 상태로 남을 수 있습니다. 프로덕션 빌드가 필요한 경우 대화형 에이전트 워크플로우 외부에서 수행하세요.

## 2. 의존성 동기화 유지

의존성을 추가하거나 업데이트할 때 다음을 기억하세요:

1. 적절한 락파일(`package-lock.json`, `pnpm-lock.yaml`, `yarn.lock`)을 업데이트합니다.
2. Next.js가 변경사항을 감지할 수 있도록 개발 서버를 재시작합니다.

## 3. 코딩 규칙

* 새로운 컴포넌트와 유틸리티에는 TypeScript(`.tsx`/`.ts`)를 선호합니다.
* 실용적인 경우 컴포넌트별 스타일은 컴포넌트와 동일한 폴더에 배치합니다.

## 4. 유용한 명령어 요약

| 명령어             | 목적                                               |
| ------------------ | -------------------------------------------------- |
| `npm run dev`      | HMR이 포함된 Next.js 개발 서버를 시작합니다.       |
| `npm run lint`     | ESLint 검사를 실행합니다.                          |
| `npm run test`     | 테스트 스위트를 실행합니다(있는 경우).             |
| `npm run build`    | **프로덕션 빌드 – _에이전트 세션 중에 실행하지 마세요_** |

---

이러한 관행을 따르면 에이전트 지원 개발 워크플로우가 빠르고 신뢰할 수 있게 유지됩니다. 의심스러운 경우 프로덕션 빌드를 실행하는 대신 개발 서버를 재시작하세요.
