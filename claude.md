# Claude를 위한 한국어 번역 프로젝트 가이드라인

이 문서는 agents.md 프로젝트의 한국어 번역 작업을 수행하는 Claude 에이전트를 위한 가이드라인입니다.

## 프로젝트 개요

agents.md는 AI 코딩 에이전트를 위한 간단하고 개방적인 형식입니다. 이 프로젝트의 목표는 모든 문서와 웹사이트 콘텐츠를 한국어로 번역하여 한국어 사용자들이 쉽게 접근할 수 있도록 하는 것입니다.

## 번역 원칙

### 1. 정확성과 자연스러움
- 원문의 의미를 정확하게 전달하되, 한국어로 자연스럽게 표현
- 기술 용어는 가능한 한 업계 표준 한국어 용어 사용
- 필요시 영문 원어를 병기 (예: "에이전트(agent)")

### 2. 일관성 유지
- 동일한 용어는 문서 전체에서 동일하게 번역
- 주요 용어 번역 기준:
  - `agent` → 에이전트
  - `coding agent` → 코딩 에이전트
  - `AI coding agent` → AI 코딩 에이전트
  - `repository` → 레포지토리
  - `development environment` → 개발 환경
  - `Hot Module Replacement (HMR)` → 핫 모듈 교체(HMR)

### 3. 파일 명명 규칙
- 한국어 번역 파일은 원본 파일명에 `.ko`를 추가
  - 예: `README.md` → `README.ko.md`
  - 예: `AGENTS.md` → `AGENTS.ko.md`

### 4. 코드 블록 및 명령어
- 코드 예제와 명령어는 번역하지 않고 원문 유지
- 코드 주석은 번역
- 명령어 설명은 번역

## 번역 작업 순서

1. **claude.md** (이 문서) - 번역 가이드라인 ✓
2. **README.md** → README.ko.md - 프로젝트 소개 페이지
3. **AGENTS.md** → AGENTS.ko.md - 이 레포지토리의 에이전트 가이드라인
4. **웹사이트 컴포넌트** (components/*.tsx) - UI 텍스트
5. **기타 문서** - 필요시 추가 문서

## 번역 시 주의사항

### 마크다운 형식 유지
- 헤더 레벨(`#`, `##`, `###`)은 동일하게 유지
- 링크 형식 `[텍스트](URL)` 유지 (텍스트만 번역, URL은 원문)
- 코드 블록 펜스(```)는 그대로 유지

### 예제 코드
- 예제 코드 내부는 번역하지 않음
- 코드 블록 위아래의 설명 텍스트는 번역

### 링크 처리
- 외부 링크 URL은 변경하지 않음
- 내부 문서 링크는 한국어 버전으로 업데이트
  - 예: `[AGENTS.md](AGENTS.md)` → `[AGENTS.md](AGENTS.ko.md)`

## 개발 서버 실행 (테스트용)

웹사이트 번역 후 확인이 필요한 경우:

```bash
npm run dev
# 또는
pnpm dev
```

브라우저에서 http://localhost:3000 접속하여 확인

## 참고사항

- 이 프로젝트는 Next.js 기반 웹사이트를 포함
- TypeScript와 React 컴포넌트 사용
- 웹사이트 번역 시 `.tsx` 파일 내의 텍스트도 번역 필요

## 질문이나 불확실한 사항

- 기술 용어 번역이 불확실한 경우: 영문 원어를 괄호 안에 병기
- 문맥상 의미가 모호한 경우: 원문의 의도를 최대한 살려 번역
- 특수한 형식이나 구조: 원문 형식을 최대한 유지
